# Project Atlas Foundation Charter

## Mission

Enable people to discover, prioritise, and collaboratively solve important problems using open coordination, public contribution, and AI-assisted delivery.

## Vision

Project Atlas aims to become an open coordination layer for collective progress: a place where problems can be proposed, evaluated, broken down, resourced, built, and delivered in the open.

## Non-negotiables

1. **Open by default** - public documents, public decisions, public roadmaps, and public outputs unless there is a clearly justified safety or privacy reason.
2. **Public-good first** - the platform should prioritise useful outcomes over growth metrics, hype, or commercial capture.
3. **Human-led, AI-assisted** - AI agents can research, plan, draft, review, and support, but final governance remains human.
4. **No paywall on community outputs** - community-created specifications, documentation, and public-good software should remain freely accessible.
5. **Transparent governance** - roadmap and moderation decisions should be recorded and explainable.
6. **Contribution over status** - influence should come from useful work, not title, employer, follower count, or seniority.
7. **Build before endless debate** - discussions should be structured toward research, decisions, prototypes, and outcomes.

## What belongs here

Project Atlas should welcome:

- Open-source software ideas
- Public-good automation projects
- Agentic workflow templates
- Research-backed proposals
- Accessibility and inclusion tools
- Climate, education, health, civic, and community-support projects
- Shared specifications that others can implement

## What does not belong here

The project should not be used for:

- Spam, scams, or token-selling schemes
- Closed-source capture of community work
- Unsafe, illegal, or harmful projects
- Harassment or culture-war campaigning
- Pure self-promotion with no public contribution

## Founding assumption

The project starts with a deliberately lightweight model using GitHub, public documentation, and community voting. Custom platform software should only be built once the operating model has been validated.
