# Contributing

Thank you for considering a contribution to Project Atlas Foundation.

This project needs more than code. Useful contributions include research, writing, moderation, design, project management, testing, architecture, documentation, accessibility review, and community support.

## Ways to contribute

- Submit a problem worth solving
- Improve an existing proposal
- Research existing solutions
- Help estimate impact and feasibility
- Turn a proposal into an implementation plan
- Create issues for a project
- Write documentation
- Build prototypes
- Review pull requests
- Improve governance and process

## Contribution workflow

1. Read the [charter](CHARTER.md).
2. Check existing issues, discussions, and backlog items.
3. For a new idea, use the proposal template.
4. For documentation changes, open a pull request.
5. For larger governance changes, open an issue first.

## Proposal quality bar

Good proposals should explain:

- The problem
- Who is affected
- Why now
- Existing solutions
- What should be built
- Risks and trade-offs
- Skills needed
- How success could be measured

## Behaviour expectations

Be constructive. Assume good intent, but prioritise clarity, safety, and usefulness. Disagreement is welcome; personal attacks are not.

## Licence expectations

By contributing documentation or process materials, you agree that your contribution may be included under the repository documentation licence. Code contributions should use the licence selected for the relevant project.
