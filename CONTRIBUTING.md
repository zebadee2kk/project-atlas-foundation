# Contributing to Project Atlas Foundation

Thank you for considering a contribution to Project Atlas Foundation.

This project needs more than code. Useful contributions include research, writing, moderation, design, project management, testing, architecture, documentation, accessibility review, and community support.

## Quick Start

1. Read the [charter](CHARTER.md) and [governance](GOVERNANCE.md)
2. Browse open [proposals](https://github.com/zebadee2kk/project-atlas-foundation/issues?q=is%3Aissue+is%3Aopen+label%3Aproposal) and [tasks](https://github.com/zebadee2kk/project-atlas-foundation/issues?q=is%3Aissue+is%3Aopen+label%3Atask)
3. Join [GitHub Discussions](https://github.com/zebadee2kk/project-atlas-foundation/discussions) for broader conversation
4. Pick something that interests you and start contributing

## Ways to Contribute

### Proposals & Research
- **Submit a proposal:** Use the [proposal template](https://github.com/zebadee2kk/project-atlas-foundation/issues/new?template=proposal.yml) to suggest a new project
- **Improve a proposal:** Add research, existing solutions, or risk analysis to an open proposal
- **Score a proposal:** Use the [impact scoring guide](docs/impact-scoring.md) to help prioritise work
- **Safety review:** Apply the [safety checklist](docs/safety-agent-checklist.md) to evaluate risks

### Documentation
- Fix typos, broken links, or unclear explanations
- Write how-to guides for common tasks
- Translate documentation into other languages
- Improve the [operating model](docs/operating-model.md) based on experience

### Code & Technical
- Build prototypes, tools, or automations
- Review pull requests
- Write tests and CI configurations
- Improve the [landing page](website/index.html)

### Community
- Welcome new contributors
- Help moderate discussions
- Organise events or working groups
- Recruit founding contributors

### Governance
- Propose improvements to the [governance model](GOVERNANCE.md)
- Participate in proposal promotion decisions
- Help define the [naming process](docs/naming-process.md)

## Contribution Workflow

### For Documentation Changes
1. Fork the repository (or branch if you have write access)
2. Make your changes
3. Open a pull request with a clear description
4. At least one reviewer must approve before merge

### For Proposals
1. Read [docs/proposer-quickstart.md](docs/proposer-quickstart.md)
2. Score your idea using [docs/impact-scoring.md](docs/impact-scoring.md)
3. Open an issue using the [proposal template](https://github.com/zebadee2kk/project-atlas-foundation/issues/new?template=proposal.yml)
4. Engage with reviewers and iterate on your proposal

### For Tasks
1. Find an open task or create one using the [task template](https://github.com/zebadee2kk/project-atlas-foundation/issues/new?template=task.yml)
2. Comment on the issue to claim it (avoid duplicate work)
3. Complete the work and open a PR or update the issue with your findings

### For Governance Changes
1. Open an issue describing the proposed change
2. Allow 7 days for community comment
3. Require two maintainer approvals before merging
4. Major changes should go through the [proposal lifecycle](docs/proposal-lifecycle.md)

## AI-Assisted Contributions

Project Atlas welcomes AI-assisted contributions. If you use AI tools:

- **Disclose it.** Note in your PR or issue that the work was AI-assisted
- **Review the output.** You are responsible for the accuracy of what you submit
- **Follow the [AI Assistant Policy](docs/ai-assistant-policy.md)** for rules on AI use
- **Don't use AI for governance decisions.** Proposals, voting, and moderation remain human-led

## Pull Request Guidelines

Every PR should include:
- **Summary:** What changed and why
- **Files modified:** List of changed files
- **Risks:** Any backward-compatibility or safety concerns
- **AI disclosure:** Whether AI tools were used

## Proposal Quality Bar

Good proposals should explain:
- The problem and who is affected
- Why now
- Existing solutions and their limitations
- What should be built or researched
- Risks and trade-offs
- Skills needed
- How success could be measured

See the [proposal template](https://github.com/zebadee2kk/project-atlas-foundation/issues/new?template=proposal.yml) for the full structure.

## Behaviour Expectations

Be constructive. Assume good intent, but prioritise clarity, safety, and usefulness. Disagreement is welcome; personal attacks are not.

See the [Code of Conduct](CODE_OF_CONDUCT.md) for the full community standards.
