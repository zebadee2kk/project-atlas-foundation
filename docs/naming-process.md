# Naming Process

Project Atlas is a working codename only. The permanent name should be selected through a deliberate process, not intuition.

## Naming criteria

The final name should be:

- Distinctive and memorable
- Easy to say and spell
- Not narrowly tied to AI, agents, code, or software
- Suitable for a foundation or public-good movement
- Available as a domain, ideally `.org`
- Available as a GitHub organisation or repository namespace
- Available as a community handle where practical
- Low-risk from a trademark perspective

## Avoid

Avoid saturated terms such as:

- AI
- Agent
- GPT
- LLM
- Open
- Build
- Hub
- Forum
- Community
- Commons
- Labs

These are either overused or make the project sound like a short-lived AI startup.

## Suggested process

1. Define the mission and positioning.
2. Generate 200+ candidate names across multiple themes.
3. Remove obvious conflicts through search.
4. Check domain availability.
5. Check GitHub, Reddit, Discord, and social handles.
6. Run trademark searches in relevant jurisdictions.
7. Shortlist 5 to 10 options.
8. Gather feedback from early contributors.
9. Choose one name and document the decision.

## Naming themes

Potential themes:

- Navigation and direction
- Coordination and alignment
- Civilisation and public infrastructure
- Shared maps, backlogs, ledgers, or operating systems
- Human + machine cooperation

## Decision rule

Do not buy a domain, create a permanent organisation, or publicly launch under a name until the checks above are complete.
