# New Project Checklist

Promoting a proposal to an active project space is a structural milestone. This checklist ensures minimum completeness and reduces avoidable setup debt.

## Required before handover
- [ ] Proposal is in `Approved` state with recorded score and rationale.
- [ ] README drafted with problem statement, scope, success metrics.
- [ ] One initial maintainer accepted responsibility and has repo access.
- [ ] At least two milestones defined.
- [ ] `.github/ISSUE_TEMPLATE/bug.yml` and `.github/ISSUE_TEMPLATE/feature.yml` copied from `best-practice-repo-template` or equivalent.
- [ ] `docs/project-lifecycle.md` acknowledged in README.
- [ ] Governance or CONTRIBUTING note added if shared work will occur.

## First 30 days
- [ ] Weekly triage cadence established.
- [ ] Two maintainer reviews completed.
- [ ] At least one community contribution merged.

## Governance expectations
- [ ] Naming follows `docs/naming-process.md`.
- [ ] Impact scoring updated for the project’s domain if needed.
- [ ] Board/Project and Discussions item created to track the project lifecycle.
